# GitHub Release Draft

Suggested tag: `v10.5.15`
Suggested title: `translatorFork_MOD v10.5.15`
Range: `8a4a302..37346b9` (`2026-04-17` - `2026-04-19`)

## Release Body

## Что нового

- Добавлен `Gemini Reader` в стартовое меню приложения: отдельный инструмент для озвучивания EPUB в общем runtime приложения.
- `Gemini Reader` получил полноценный TTS-пайплайн с режимами `Live API` и `Flash TTS`, AI-подготовкой сценария, режимами одного и двух голосов, тестом голоса, переозвучкой выбранных глав, live-воспроизведением и экспортом MP3/видео.
- Для локальных провайдеров добавлено автообнаружение моделей из Ollama и LM Studio с ручным обновлением списка моделей прямо в настройках.
- Настройки сессии перевода теперь сохраняются между перезапусками и возвратом в главное меню: восстанавливаются провайдер, модель, число воркеров, batching/chunking, лимиты и другие ключевые параметры сессии.
- Исправлено восстановление числа воркеров в AI-генерации глоссария после загрузки сохранённых настроек.
- Глоссарий теперь стабильно сортируется по полю `original`, а пустые строки остаются в конце списка.

## Исправления

- Исправлен вызов локальных и синхронных обработчиков в `Consistency Checker`, из-за которого local-like провайдеры могли обрабатываться некорректно.
- Улучшен предпросмотр в `Consistency Checker`: включён корректный перенос длинных строк и более контрастная подсветка изменённых строк.
- Предпросмотр главы из очереди теперь открывает самый свежий итоговый файл проекта и предпочитает `validated`-версию, если она уже существует.
- Исправлен возврат в главное меню из `RanobeLib Uploader`.
- Исправлена обработка `mem://` путей в `os_patch` для `exists`, `isfile` и `isdir`, что убирает часть проблем с виртуальной ФС и вспомогательным runtime.

## Технические изменения

- В сборки и launch-скрипты интегрированы зависимости для `Gemini Reader`, включая `ffmpeg` и `ffprobe`.
- Обновлён `README` и расширено тестовое покрытие для TTS-сценариев, локального discovery моделей, сохранения настроек, consistency preview и сортировки глоссария.

## Важно

- В репозитории пока нет тегов, поэтому этот релиз собран по всему диапазону от первого импорт-коммита до текущего `HEAD`.
- Breaking changes в этом диапазоне не выделены.
- Если приложение запускается из исходников, стоит обновить зависимости из `requirements.txt` перед использованием новых Reader/TTS-возможностей.
