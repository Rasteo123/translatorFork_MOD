# Gemini EPUB Translator

**Gemini EPUB Translator** — это профессиональный программный комплекс для пакетного перевода EPUB книг с использованием современных AI моделей. Приложение предоставляет мощный графический интерфейс на PyQt6 и включает в себя набор инструментов для всех этапов работы: от подготовки и перевода до проверки качества и финальной сборки книги.

Вместо старых режимов ротации и параллельных окон, программа теперь использует единый, многопоточный интерфейс, который обеспечивает максимальную производительность и гибкость управления процессом.

## Ключевые возможности

### 1. Движок перевода: Скорость и контроль

Основной модуль, созданный для максимальной производительности и гибкости.

-   **Настоящая многопоточность**: Программа запускает независимые фоновые обработчики (воркеры) для каждого API-ключа, обеспечивая параллельный перевод глав в рамках единого интерфейса.
-   **Умное управление API-ключами**: Задайте пул ключей для сессии, и программа сама распределит нагрузку. Она отслеживает лимиты, ставит ключи на временную "паузу" при ошибках и автоматически возвращает их в работу после истечения тайм-аута.
-   **Мульти-API**: Используйте лучшие модели от **Google Gemini**, **OpenRouter** (включая бесплатные модели) и **Zhipu AI (GLM)**, переключаясь между ними в любой момент.
-   **Пакетная обработка и чанкинг**: Переводите целые книги. Слишком большие главы могут быть автоматически разделены на части (чанки) для обхода ограничений API и собраны обратно после перевода.
-   **Полный контроль над промптом**: Настраивайте инструкции для AI с помощью удобного редактора с поддержкой пресетов.

### 2. Менеджер глоссариев: Интеллектуальный ассистент

Это не просто редактор, а полноценный инструмент для достижения идеальной целостности терминологии.

-   **AI-ассистент для разрешения конфликтов**: Главная особенность модуля. Доверьте AI анализ всего глоссария. Он сам найдет проблемные группы терминов (прямые, обратные конфликты, наложения) и предложит готовые, логически выверенные исправления.
-   **Режим полного контроля**: Для сложных случаев используйте пошаговый "Мастер", который проведет вас через все найденные проблемы, предлагая варианты решения на каждом этапе.
-   **Мастер импорта**: Забудьте о ручной конвертации. Встроенный мастер позволяет настроить правила импорта для **любых текстовых форматов** (`.txt`, `.csv`) и мгновенно преобразовать их в структуру глоссария.
-   **Грамматический помощник**: Интеграция с Pymorphy помогает быстро сгенерировать грамматические примечания (род, число, склонение), чтобы помочь в обеспечении единообразия.

### 3. Интерактивный Валидатор: Профессиональная вычитка

Мощный инструмент для финальной доработки и проверки качества, превращающий вычитку в удобный процесс.

-   **Сравнение "бок о бок"**: Просматривайте оригинальный и переведенный текст в двух синхронизированных панелях.
-   **Два режима подсветки**:
    -   **Режим кода**: Анализируйте HTML-структуру с полноценной подсветкой синтаксиса тегов, атрибутов и значений.
    -   **Режим пунктуации**: Мгновенно находите ошибки в оформлении диалогов с помощью яркой подсветки тире, кавычек-«ёлочек» и других знаков.
-   **Автоматические проверки**: Валидатор сам ищет и помечает главы с распространенными проблемами: нарушение XML-структуры, потеря тегов, непереведенный текст, аномальное соотношение длин и т.д.
-   **Редактирование на месте**: Вносите правки прямо в окне валидатора. Сохраняйте изменения и помечайте главы как **"Готовые"** для финальной сборки.

### 4. Менеджер проектов и Сборщик EPUB: Финальный штрих

Инструменты, которые связывают все этапы работы в единый, надежный цикл.

-   **Надёжная система проектов**: Программа создает файл-карту `translation_map.json`, который отслеживает **все версии переводов** для каждой главы, позволяя легко переключаться между ними или повторять перевод с другими моделями.
-   **Мощный сборщик EPUB с двумя режимами**:
    -   **Создание с нуля**: Соберите новую книгу из переведенных глав, указав метаданные.
    -   **🔥 Обновление оригинала**: Уникальная функция, которая заменяет текстовые блоки в исходном EPUB, **сохраняя 100% оригинального форматирования: стили, шрифты, обложку и все изображения.**

### ⚙️ Системные инструкции (System Prompt)

Это продвинутая функция, позволяющая задать **постоянную, высокоуровневую инструкцию** для AI, которая будет действовать на протяжении всей сессии перевода. Она определяет **роль, личность или набор правил**, которым модель должна неукоснительно следовать.

    1.  **Как это работает?** Системная инструкция отправляется отдельно от основного промпта (который содержит текст главы, глоссарий и т.д.). Это позволяет AI лучше разделять свою "личность" и текущую "задачу".

    2.  **Когда это полезно?**
    -   Для задания **стиля перевода**: *"Ты — профессиональный литературный переводчик, специализирующийся на фэнтези. Твой стиль — богатый и образный, избегай канцеляризмов."*
    -   Для установки **тона**: *"Всегда сохраняй формальный и уважительный тон."*
    -   Для определения **ограничений**: *"Никогда не используй в тексте современные сленговые выражения."*

    3.  **Преимущество:** Это освобождает основной промпт от "воды" и общих инструкций, позволяя сосредоточить его только на задаче перевода конкретного фрагмента, что повышает точность и стабильность результата.


### 🧪 Пробный запуск (Dry Run): Ручное управление

Это мощный инструмент для отладки и тестирования, который позволяет увидеть **финальный промпт**, отправляемый AI, и вручную задать ответ модели.

-   **Как это работает?**
    1.  Нажмите кнопку **"🧪 Пробный запуск"** в главном окне.
    2.  Откроется окно, где будет показан полный текст промпта, включая глоссарий, контекст и сам текст главы.
    3.  Нажмите кнопку **"📋 Копипаст"**, чтобы скопировать весь этот текст в буфер обмена.
    4.  Теперь вы можете использовать этот промпт где угодно: вставить его в веб-интерфейс Google AI Studio, OpenRouter или любой другой сервис для проверки реакции модели.
    5.  Получив ответ от модели, вернитесь в окно пробного запуска и нажмите кнопку **"✍️ Ручной Ответ"**.
    6.  Вставьте полученный перевод в текстовое поле и нажмите **"✅ Применить как перевод"**.

-   **Когда это полезно?**
    -   Для **тестирования промптов**: быстро проверяйте, как разные инструкции влияют на результат, не запуская полную сессию.
    -   Для **использования внешних моделей**: переводите главу с помощью модели, которой нет в программе, и интегрируйте результат.
    -   Для **отладки**: если перевод получается некорректным, вы можете увидеть, что именно "видит" AI, и скорректировать инструкции или глоссарий.


## Установка

1.  **Клонируйте репозиторий**:
    ```bash
    git clone https://your-repository-url/gemini_translator_project.git
    cd gemini_translator_project
    ```

2.  **Создайте и активируйте виртуальное окружение** (рекомендуется):
    ```bash
    # Для Windows
    python -m venv venv
    venv\Scripts\activate

    # Для macOS/Linux
    python3 -m venv venv
    source venv/bin/activate
    ```

3.  **Установите зависимости**:
```bash
    pip install -r requirements.txt
```

## Первый запуск и настройка

При первом запуске программа попросит вас ввести API ключи.

-   **Google Gemini**: Ключи можно получить в [Google AI Studio](https://aistudio.google.com/app/apikey).
-   **OpenRouter**: Ключ можно получить на сайте [OpenRouter.ai](https://openrouter.ai/keys).
-   **GLM**: Ключ можно получить на платформе [Zhipu AI](https://open.bigmodel.cn/).

Введенные ключи сохраняются в конфигурационном файле в домашней директории пользователя (`~/.epub_translator/settings.json`) и будут доступны при последующих запусках во всех инструментах.

## Использование

Для запуска приложения выполните команду в корневой папке проекта:

```bash
python main.py
```


## Рабочий процесс

Программа состоит из трех независимых, но взаимосвязанных инструментов. При запуске вам будет предложено выбрать, с чего вы хотите начать. Вы можете использовать только тот инструмент, который вам нужен в данный момент.

---

### 🚀 Основной сценарий: Перевод книги

Это самый частый сценарий, который включает в себя полный цикл от начала до конца.

**Шаг 1: Создание проекта**

Проект — это связка из вашего `EPUB-файла` и `папки` для результатов.

1.  В стартовом окне выберите **"🚀 Переводчик EPUB"**.
2.  Откроется главное окно настройки. В верхней части найдите две большие кнопки:

    *   Нажмите **"Выбор файла"** и укажите путь к вашему исходному `.epub` файлу.
    *   Нажмите **"Выбор папки"** и выберите (или создайте) пустую папку, куда будут сохраняться все переводы и настройки.

3.  Как только вы выберете и файл, и папку, программа создаст в ней файл `translation_map.json`. Ваш проект готов!

**Шаг 2: Выбор глав для перевода**

После выбора файла автоматически откроется окно со списком всех HTML-глав внутри книги.

1.  По умолчанию выбраны все главы. Вы можете снять галочки с тех, которые переводить не нужно (например, оглавление или титульные страницы).
2.  Если вы работаете с уже существующим проектом, используйте кнопки-фильтры вверху (**"Только непереведенные"**, **"Непроверенные"**), чтобы быстро выбрать только нужные главы.
3.  Нажмите **"Принять"**, чтобы подтвердить свой выбор.

### Шаг 3: Детальная настройка сессии

Теперь вы находитесь в главном окне, где сосредоточены все параметры перевода. Пройдитесь по основным панелям, чтобы настроить сессию под вашу задачу.

**Панель 1: API и Параллельная обработка**

Это ядро управления скоростью и стоимостью перевода.

1.  **Провайдер API**: Выберите сервис, который будет выполнять перевод (Google Gemini, OpenRouter и т.д.). Для каждого сервиса используется свой, независимый набор ключей.

2.  **Управление ключами**: Здесь вы формируете "команду" для текущей сессии.

    *   Слева (`Доступные ключи`) — ваш общий пул ключей для выбранного провайдера.
    *   Справа (`Активные ключи для сессии`) — ключи, которые будут работать **именно сейчас**.
    *   Используйте кнопки **`>`** и **`<`** для перемещения ключей между списками. Вы можете задействовать несколько ключей одновременно для распределения нагрузки.

3.  **Обработчики**: Указывает, сколько **параллельных обработчиков** запускать. Каждый обработчик использует один уникальный API-ключ из списка `Активные ключи для сессии`.

-   **Уровень 1 (Параллелизм по ключам)**: Эта настройка определяет, сколько API-ключей работают одновременно. Поскольку у каждого ключа свои независимые лимиты запросов, увеличение этого числа — основной способ масштабирования скорости перевода.
-   **Уровень 2 (Параллелизм по запросам)**: Каждый отдельный обработчик работает асинхронно. Он может отправлять **несколько запросов одновременно** к своему личному ключу, не дожидаясь ответа на предыдущие. Это количество настраивается в панели «Настройки модели» (`Параллельные запросы`).
-   **Правило**: Количество обработчиков не может превышать количество `Активных ключей для сессии`.
-   **Баланс**: Увеличение числа обработчиков кратно увеличивает скорость, но так же кратно расходует суммарные лимиты всех задействованных ключей.



**Панель 2: Глоссарий и Контекст**

Это главный инструмент для обеспечения единообразия перевода имен, названий и уникальных терминов.

1.  Нажмите **"Импорт…"**, чтобы загрузить готовый глоссарий из файла (`.json`, `.txt`), или **"➕ Добавить"**, чтобы внести термины вручную.
2.  Для глубокого анализа, поиска конфликтов и автоматического исправления используйте отдельный инструмент — **"🛠️ Менеджер…"**.



**Панель 3: Промпт (Инструкции для AI)**

Здесь вы задаете **роль и правила** для AI, управляя стилем и качеством перевода.

1.  Вы можете оставить стандартный промпт, который оптимизирован для большинства задач.
2.  Используйте **пресеты** для быстрого переключения между разными стилями (например, "Технический перевод", "Художественный фэнтези") или напишите свой собственный.



**Панель 4: Оптимизация (Режим обработки)**

Эти опции позволяют оптимизировать то, как главы отправляются на перевод, экономя время и токены.

1.  **Пакетная обработка (Batching)**: Объединяет несколько маленьких глав в один большой запрос к AI. Идеально для книг с множеством коротких глав (например, системных сообщений в литРПГ).
2.  **Авто-чанкинг (Chunking)**: Автоматически разделяет одну очень большую главу на несколько частей (чанков) для обхода ограничений API. После перевода части автоматически собираются обратно.



**Панель 5: Настройки модели**

Здесь собраны тонкие настройки, влияющие на «поведение» AI.

1.  **Модель**: Выберите конкретную AI-модель от выбранного провайдера (например, `gemini-2.5-flash` — быстрая и дешевая, `gemini-2.5-pro` — более мощная).
2.  **Температура**: Регулирует «креативность» AI.

    *   Меньше `1.0` — более строгий и дословный перевод.
    *   Значение `1.0` — сбалансированный вариант.
    *   Больше `1.0` — более вольный и художественный перевод.

### Шаг 4: Запуск перевода

1.  Когда все настроено, нажмите большую зеленую кнопку **"🚀 Старт"**.
2.  Процесс перевода начнется в фоновом режиме. Вы можете следить за ним:

    *   Во вкладке **"Список Задач"** будет обновляться статус каждой главы в реальном времени (Ожидание -> В работе -> Успешно/Ошибка).
    *   Во вкладке **"Логирование"** будут появляться подробные технические сообщения о ходе процесса.

3.  Для остановки в любой момент нажмите **"❌ Стоп"**.

---

### ✅ Проверка и доработка готового перевода

Этот сценарий идеален, когда перевод уже завершен, и вы хотите провести вычитку.

1.  В стартовом окне выберите **"✅ Валидатор переводов"**.
2.  Появится диалог, предлагающий два варианта:

    *   **"Загрузить проект из истории"**: Самый простой способ. Выберите ваш проект из списка недавних.
    *   **"Выбрать папку и файл вручную"**: Укажите папку с результатами и оригинальный `.epub` файл.

3.  Откроется **Интерактивный Валидатор**, где вы сможете построчно сравнивать оригинал и перевод, переключаться в режим кода и вносить правки.



### 📚 Работа с терминами в Менеджере глоссариев

Используйте этот инструмент для подготовки "чистого" глоссария перед переводом или для анализа уже существующего.

1.  В стартовом окне выберите **"📚 Менеджер глоссариев"**.
2.  Откроется мощное окно-редактор.
3.  Нажмите **"Открыть глоссарии…"** и загрузите один или несколько файлов (`.json`, `.txt`).
4.  Нажмите **"🔄 Анализ"**, чтобы программа нашла все типы конфликтов.
5.  Используйте кнопку **"🤖 Исправить с AI…"** для автоматического разрешения проблем или кнопки **"Прямые конфликты"**, **"Наложения"** и т.д., чтобы пройтись по ним в ручном режиме с помощью пошагового мастера.



## Структура проекта

Проект организован в пакет `gemini_translator` для лучшей модульности:

-   `gemini_translator/api/`: Логика работы с внешними API и конфигурации моделей.
-   `gemini_translator/core/`: Основная логика перевода (воркеры, сессии, менеджеры задач).
-   `gemini_translator/ui/`: Все компоненты пользовательского интерфейса (главное окно, диалоги, виджеты).
-   `gemini_translator/utils/`: Вспомогательные функции и классы (работа с EPUB, текстом, глоссарями).
-   `main.py`: Точка входа в приложение, запускающая стартовый диалог.



## ⚙️ Использование локальных моделей (Ollama, LM Studio и др.)

Приложение поддерживает работу с большими языковыми моделями, запущенными непосредственно на вашем компьютере. Это обеспечивает полную конфиденциальность, отсутствие лимитов на запросы и нулевую стоимость использования API.

Для этого используется стандартный, OpenAI-совместимый API, который предоставляют популярные серверы, такие как **Ollama** или **LM Studio**.

Ниже приведена инструкция по настройке на примере **Ollama** как самого простого и популярного решения.

### Шаг 1: Установка локального сервера

1.  **Скачайте и установите Ollama** с [официального сайта](https://ollama.com/).
2.  После установки Ollama будет работать в фоновом режиме, готовая к приему запросов.

### Шаг 2: Загрузка модели

1.  Откройте терминал (командную строку) на вашем компьютере.
2.  Выполните команду для скачивания нужной модели. Например, для Llama 3 8B:
    ```bash
    ollama run llama3:8b
    ```
    Или для более легковесной Phi-3 Mini:
    ```bash
    ollama run phi3:mini
    ```
3.  Дождитесь полного скачивания модели. После этого вы можете закрыть терминал.

> **Важно:** Имя модели, которое вы используете в команде (`llama3:8b`, `phi3:mini`), — это её **ID**, который понадобится на следующем шаге.

### Шаг 3: Настройка и активация локального провайдера

1.  Найдите и откройте файл `api_providers.json` в папке `config` вашего приложения.
2.  Найдите в файле секцию `"local"`. По умолчанию она выглядит так и **скрыта** из интерфейса:

    ```json
    "local": {
        "display_name": "Локальные модели (Ollama/LM Studio)",
        "handler_class": "LocalApiHandler",
        "base_url": "http://127.0.0.1:11434/v1/chat/completions",
        "visible": false,
        "is_async": true,
        // ... прочие параметры
        "models": {
            "Llama 3 8B (локально)": {"id": "llama3:8b", "rpm": 1000, "needs_chunking": true, "max_output_tokens": 8192},
            "Phi-3 Mini 4k (локально)": {"id": "phi3:mini", "rpm": 1000, "needs_chunking": true, "max_output_tokens": 4096}
        }
    }
    ```

3.  **✅ Активируйте провайдер.**
    Чтобы опция "Локальные модели" появилась в программе, измените строку:
    `"visible": false,`
    на
    `"visible": true,`

4.  **Проверьте адрес сервера (`base_url`):**
    *   Для **Ollama** адрес по умолчанию: `http://127.0.0.1:11434/v1/chat/completions`.
    *   Для **LM Studio** адрес обычно: `http://127.0.0.1:1234/v1/chat/completions`.
    Если вы используете LM Studio или другой сервер, измените `base_url` на правильный адрес.

5.  **Добавьте свою модель в список `models`:**
    *   **`"Имя в интерфейсе"`**: Придумайте любое понятное имя, которое будет отображаться в списке моделей (например, `"My Llama 3 70B"`).
    *   **`"id"`**: Укажите тот самый ID модели из **Шага 2** (например, `"llama3:70b"`).

После сохранения файла `api_providers.json` и перезапуска приложения новый провайдер станет доступен для выбора.

### Шаг 4: Запуск перевода

1.  Запустите приложение.
2.  В выпадающем списке **"Провайдер API"** выберите **"Локальные модели (Ollama/LM Studio)"**.
3.  В списке **"Модель"** выберите ту, которую вы добавили.
4.  В списке **"Ключ API"** введите любой случайный ключ.

> ⚠️ **Важно:** Для локального провайдера API ключ **не используется и никуда не отправляется**. Выбор ключа в интерфейсе — это формальность, необходимая для запуска процесса.

Готово! Теперь приложение будет отправлять запросы на вашу локальную модель.


### 🚀 Продвинутая настройка: несколько локальных серверов:

Вы можете одновременно использовать модели с разных серверов (например, одна через Ollama, другая — через LM Studio). Для этого просто добавьте параметр `base_url` прямо в конфигурацию конкретной модели в файле `api_providers.json`.

**Пример**:

```json
    "local": {
        "display_name": "Локальные модели",
        // Адрес по умолчанию для всех моделей этого провайдера (Ollama)
        "base_url": "http://127.0.0.1:11434/v1/chat/completions",
        // ...
        "models": {
            "Llama 3 (Ollama)": {
                "id": "llama3:8b", 
                "rpm": 1000
            },
            "Gemma 2 (LM Studio)": {
                "id": "google/gemma-2-9b-it-gguf", 
                "rpm": 1000,
                // Персональный адрес для этой модели, переопределяет основной
                "base_url": "http://127.0.0.1:1234/v1/chat/completions"
            }
        }
    }
```
