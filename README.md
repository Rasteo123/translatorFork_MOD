# translatorFork_MOD

`translatorFork_MOD` — это настольное приложение на PyQt6 для перевода EPUB-книг, работы с глоссариями, валидации результата, экспорта в Rulate Markdown и загрузки глав на RanobeLib.

Важно: внутри кода местами сохранены исторические имена `Gemini EPUB Translator` и пакет `gemini_translator`. Для этого форка это нормальное состояние.

## Что умеет приложение

- Переводить EPUB-книги пакетно, через очередь задач и выбор нужных глав.
- Работать с несколькими API-провайдерами и локальными моделями.
- Делить слишком большие главы на чанки и собирать результат обратно.
- Сохранять проектное состояние и восстанавливать очередь после прерывания.
- Проверять перевод в отдельном валидаторе с ручной правкой.
- Поддерживать отдельный менеджер глоссариев с AI-анализом конфликтов.
- Экспортировать EPUB в Rulate Markdown, разбивать главы и запускать загрузчик RanobeLib.

## Инструменты в стартовом меню

При запуске приложения сейчас доступны такие инструменты:

1. `Переводчик EPUB`
   Основной режим перевода EPUB-проектов.
2. `Валидатор переводов`
   Ручная вычитка, исправление и отметка готовности глав.
3. `Менеджер глоссариев`
   Отдельный редактор глоссариев, импорт, анализ и AI-очистка.
4. `EPUB -> Rulate MD`
   Конвертация EPUB в Markdown-формат под Rulate.
5. `Сплиттер глав`
   Разделение больших глав для EPUB и Rulate Markdown.
6. `RanobeLib Uploader`
   Загрузка на RanobeLib, логин через браузер и вспомогательные Rulate-сценарии.

## Актуальные провайдеры перевода

Сейчас в [`config/api_providers.json`](./config/api_providers.json) видимыми включены:

- `Google Gemini Free`
- `Deepseek API`
- `Perplexity API MOD`
- `OpenRouter (бесплатные модели)`
- `Hugging Face (Free)`
- `Локальные модели (Ollama/LM Studio)`
- `NeuroAPI (Платный / VIP Модели)`
- `ChatGPT Web (work_ascii)`

Точный список моделей меняется через конфиг [`config/api_providers.json`](./config/api_providers.json).

## Ключевые возможности

### Переводчик EPUB

- Очередь глав и пакетный запуск.
- История проектов и быстрое повторное открытие.
- Сохранение и восстановление snapshot-файла очереди.
- Автоматический чанкинг длинных глав.
- Пресеты промптов и редактируемый пользовательский промпт.
- Выбор базовых шаблонов для AI-глоссария.
- Автоматический AI-consistency после перевода.
- Ручной перевод и dry-run сценарии.
- Поддержка прокси на уровне сетевых клиентов.

### Валидатор

- Сравнение оригинала и перевода бок о бок.
- Проверка HTML-структуры.
- Правка текста прямо в окне валидатора.
- Повторное открытие проектов из истории или по ручному выбору путей.

### Менеджер глоссариев

- Отдельный запуск или работа с проектным глоссарием.
- AI-поиск и исправление конфликтов терминов.
- Мастер импорта внешних текстовых источников.
- Частотный и остаточный анализ.
- Морфологические подсказки через `pymorphy3`.

### Дополнительные инструменты

- `EPUB -> Rulate MD`: выбор глав, переименование, правила томов, экспорт в один или несколько Markdown-файлов.
- `Сплиттер глав`: деление больших глав с сохранением структуры и логики именования.
- `RanobeLib Uploader`: браузерный логин, получение списка глав с Rulate и подготовка к загрузке на RanobeLib.

## Быстрый старт

### Windows

Запустите:

```bat
run.bat
```

### Linux / macOS

Запустите:

```bash
chmod +x run.sh
./run.sh
```

Оба скрипта сами создают локальную `.venv`, обновляют `pip`, ставят зависимости из [`requirements.txt`](./requirements.txt) и запускают приложение.

## Ручной запуск

Если нужен полностью ручной сценарий:

```bash
git clone https://github.com/primalrin/translatorFork_MOD.git
cd translatorFork_MOD
python -m venv .venv
```

Windows:

```bat
.venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python main.py
```

Linux / macOS:

```bash
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python main.py
```

## Дополнительные требования для отдельных режимов

Некоторые инструменты требуют дополнительных локальных компонентов:

- Для Playwright-режимов вроде `ChatGPT Web (work_ascii)` и `RanobeLib Uploader` один раз установите Chromium:

```bash
python -m playwright install chromium
```

- Для `ChatGPT Web (work_ascii)` нужен `node` в `PATH`.
- На Linux может понадобиться пакет `python3-venv`, если он не установлен системой по умолчанию.

## Первый запуск

При первом запуске обычно нужно:

1. Открыть `Переводчик EPUB`.
2. Выбрать исходный `.epub` и папку проекта.
3. Отметить главы для обработки.
4. Настроить провайдера, модель, промпт, глоссарий и параметры очереди.
5. Запустить перевод.

Глобальные настройки приложения хранятся в пользовательской папке `.epub_translator`.

## Типовой рабочий сценарий

1. Откройте `Переводчик EPUB`.
2. Выберите исходный EPUB и папку проекта.
3. Соберите очередь по нужным главам.
4. Настройте провайдера, модель, промпт и правила работы с глоссарием.
5. Запустите перевод.
6. При необходимости сохраните snapshot очереди или восстановите его после сбоя.
7. Откройте `Валидатор переводов` и доведите результат вручную.
8. Экспортируйте итоговый материал или используйте дополнительные инструменты проекта.

## Какие файлы появляются в проекте

Во время работы в папке проекта могут появляться, например:

- `translation_map.json`
- `queue_snapshot.db`
- переведенные главы
- данные глоссария и промежуточные файлы проекта

Точный состав зависит от выбранного сценария работы.

## Структура репозитория

- [`main.py`](./main.py): точка входа и маршрутизация по инструментам.
- [`gemini_translator/`](./gemini_translator/): основной пакет приложения.
- [`config/`](./config/): провайдеры, промпты, внутренние шаблоны и базовые глоссарии.
- [`ranobelib/`](./ranobelib/): встроенные исходники загрузчика RanobeLib.
- [`tests/`](./tests/): регрессионные тесты для глоссария, валидатора, очереди, истории проектов и runtime-логики.
- [`run.bat`](./run.bat), [`run.sh`](./run.sh): рекомендуемые скрипты запуска.
- [`build.bat`](./build.bat), [`build_master.py`](./build_master.py): сборочные скрипты.

## Сборка

Зависимости для запуска перечислены в [`requirements.txt`](./requirements.txt).

Сборочные зависимости ставятся отдельно через сборочные скрипты и включают PyInstaller-стек. Если вам нужен только запуск из исходников, `requirements.txt` достаточно.

## Примечания

- Лимиты, квоты, браузерные сценарии и стабильность зависят от внешних сервисов.
- Список провайдеров и моделей задается конфигом и может меняться без переписывания кода.
- Это форк с собственным брендингом, поэтому часть старых имен в кодовой базе сохранена намеренно.
