# Прокси и сетевые проблемы

Этот файл нужен для случаев, когда программа не может достучаться до API, сайтов или браузерных сценариев из-за сети, региона или ограничений окружения.

## Когда сюда идти

Открывайте эту справку, если:

- API не отвечает;
- сервисы недоступны из вашего региона;
- браузерные режимы не запускаются;
- `ChatGPT Web` или `RanobeLib Uploader` работают нестабильно;
- программа жалуется на геоблок или недостающие зависимости.

## Настройки прокси

В окне прокси обычно можно:

- включить или выключить прокси;
- выбрать тип;
- указать хост;
- указать порт;
- ввести пользователя и пароль;
- выбрать прокси из сохранённых;
- удалить ненужный прокси.

Поддерживаются обычные варианты вроде:

- `SOCKS5`;
- `SOCKS4`;
- `HTTP`.

## Как настроить прокси

1. Откройте настройки прокси из стартового меню или соответствующей кнопки.
2. Введите тип, хост и порт.
3. Если нужно, добавьте логин и пароль.
4. Включите флажок использования прокси.
5. Сохраните настройки.
6. Перезапустите проблемный сценарий.

## Что такое сохранённые прокси

Программа умеет хранить список уже введённых прокси. Это полезно, если вы:

- переключаетесь между несколькими серверами;
- тестируете разные варианты;
- не хотите каждый раз вводить всё заново.

## Если программа пишет про `PySocks`

Для работы прокси может требоваться библиотека `PySocks`. Если программа сообщает, что прокси не может быть применён без неё, это значит:

- настройки вы ввели правильно;
- но окружение не готово к использованию прокси;
- надо установить недостающую зависимость.

## Геоблоки

Если сервис ограничивает доступ по региону, программа может показать специальное сообщение о блокировке. Для пользователя это означает:

- проблема не обязательно в ключе;
- сначала стоит проверить региональную доступность;
- прокси в таком случае часто является единственным практическим решением.

## Браузерные режимы

Для некоторых функций нужен встроенный браузерный стек:

- `ChatGPT Web (work_ascii)`;
- `RanobeLib Uploader`;
- часть сценариев с Playwright.

Обычно для них заранее нужен Chromium:

```bash
python -m playwright install chromium
```

## Когда нужен `node`

Для `ChatGPT Web (work_ascii)` дополнительно нужен `node` в `PATH`. Если его нет, браузерный сценарий может не стартовать или работать неполноценно.

## Что делать, если всё тормозит или отваливается

Проверяйте по порядку:

1. Есть ли доступ к интернету без программы.
2. Работает ли сам сервис у вас в регионе.
3. Правильный ли прокси выбран.
4. Не упёрся ли ключ в лимит.
5. Не слишком ли высокая параллельность выставлена.
6. Установлены ли Chromium и `node`, если нужен браузерный режим.

## Частые симптомы и их смысл

- Частые `NETWORK`-ошибки: проблемы связи, прокси или лимитов.
- Сервис виден, но нестабилен: возможно, слишком агрессивная параллельность.
- Сайт не открывается в браузерном режиме: проверьте Chromium и авторизацию.
- Ключи “умирают” слишком быстро: проверьте RPM и суточные лимиты.

## Практический совет

Если вы настраиваете новую сеть, не запускайте сразу целую книгу. Сначала проверьте:

- одну-две главы в переводчике;
- один короткий вход в RanobeLib;
- один короткий сценарий в Gemini Reader.

Так легче понять, что именно ломается: сеть, ключи, модель или сам рабочий процесс.
