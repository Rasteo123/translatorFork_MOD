# EPUB -> Rulate MD

`EPUB -> Rulate MD` нужен для подготовки книги к публикации на Rulate. Он не переводит текст, а конвертирует содержимое EPUB в Markdown-структуру, удобную для дальнейшей выкладки.

## Когда его использовать

Открывайте этот инструмент, если вам нужно:

- выгрузить главы из EPUB в `.md`;
- подготовить список глав для Rulate;
- переименовать главы перед экспортом;
- отметить платные главы;
- разбить результат на несколько Markdown-файлов;
- задать правила томов.

## Что нужно на входе

- EPUB-файл;
- понимание, какие главы вы хотите экспортировать;
- при необходимости — правила томов и платности.

## Что можно настроить

В окне обычно доступны такие блоки:

- выбор исходного EPUB;
- список глав;
- редактор имён глав;
- вкладка платности;
- вкладка томов;
- вкладка экспорта.

## Работа со списком глав

После выбора EPUB программа показывает список глав. Пользователь может:

- выбрать все главы;
- снять выбор;
- отметить только нужные главы;
- переименовать главу двойным кликом.

Подсказка в интерфейсе указывает на стандартные способы множественного выбора:

- `Shift + клик` для диапазона;
- `Ctrl + клик` для точечного выбора.

## Редактор имён

Вкладка редактора имён нужна, если:

- названия глав в EPUB слишком сырой вид;
- нужно массово подправить нумерацию;
- вы хотите подготовить список для AI и затем применить изменения обратно.

Обычно в процессе есть два шага:

1. `Получить список для AI`
2. `Применить изменения`

Эта схема удобна, когда вы хотите быстро привести названия к единому формату.

## Платность

Во вкладке платности можно отметить, какие главы должны считаться платными. Это полезно, если вы готовите структуру публикации заранее и хотите не расставлять всё вручную уже на сайте.

## Тома

Если в книге есть деление на тома, можно задать правила вида:

- с какой главы начинается том;
- какие главы относятся к определённому блоку;
- как должен выглядеть итоговый экспорт.

Это особенно полезно на длинных романах, где публикация идёт по частям.

## Разбиение на файлы

Вкладка экспорта позволяет решить, сколько глав попадёт в один Markdown-файл. Это удобно в двух случаях:

- вы хотите один большой файл на всю книгу;
- вы хотите несколько файлов фиксированного размера.

Разделение по файлам облегчает публикацию и ручную проверку.

## Предпросмотр результата

Перед окончательной конвертацией обычно можно посмотреть, как будет выглядеть итоговый результат. Пользуйтесь этим, если:

- боитесь потерять структуру;
- сомневаетесь в названиях;
- не уверены, как применились правила томов и платности.

## Рекомендуемый порядок работы

1. Выберите EPUB.
2. Отметьте главы.
3. При необходимости переименуйте названия.
4. Отметьте платность.
5. Задайте тома.
6. Настройте разбиение на файлы.
7. Проверьте предпросмотр.
8. Нажмите конвертацию.

## Когда этот инструмент особенно полезен

- после завершённого перевода;
- когда надо быстро получить Markdown для публикации;
- если названия глав в исходном EPUB неудобны для сайта;
- если вы ведёте публикацию по частям и томам.

## Частые ошибки

- экспорт до финальной вычитки;
- отсутствие проверки имён глав;
- случайное смешение платных и бесплатных глав;
- слишком крупный итоговый Markdown, если сайт удобнее кормить порциями.
