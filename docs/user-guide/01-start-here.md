# С чего начать

Этот файл нужен для первого знакомства с программой. Он помогает понять, что подготовить до запуска и в каком порядке осваивать основные инструменты.

## Для чего подходит программа

Приложение рассчитано на такие задачи:

- перевод EPUB-книг через AI-провайдеров и локальные модели;
- поддержка проектного глоссария;
- ручная проверка и правка перевода;
- экспорт в формат Rulate Markdown;
- загрузка глав на RanobeLib;
- озвучивание EPUB в MP3 через Gemini Reader.

## Что подготовить заранее

Перед работой желательно иметь:

- исходный `.epub`;
- отдельную папку проекта для этой книги;
- хотя бы один рабочий AI-провайдер и его API-ключ;
- понимание, нужен ли вам прокси;
- свободное место на диске под проект, временные файлы и результат.

Дополнительно для отдельных режимов:

- `python -m playwright install chromium` для браузерных сценариев;
- `node` в `PATH` для `ChatGPT Web (work_ascii)`;
- прокси, если API или сайт блокируется по региону.

## Что вы видите после запуска

После старта открывается меню выбора инструмента. В нём обычно доступны:

- `Переводчик EPUB`;
- `Валидатор переводов`;
- `Менеджер глоссариев`;
- `EPUB -> Rulate MD`;
- `Сплиттер глав`;
- `Gemini Reader`;
- `RanobeLib Uploader`.

Если не уверены, что выбрать:

- начинайте с `Переводчик EPUB`, если книги ещё нет в проекте;
- открывайте `Валидатор`, если перевод уже получен и его нужно проверить;
- открывайте `Менеджер глоссариев`, если хотите отдельно привести термины в порядок.

## Базовый порядок работы

Обычно рабочий процесс выглядит так:

1. Открыть `Переводчик EPUB`.
2. Выбрать исходный EPUB и папку проекта.
3. Отметить главы для обработки.
4. Настроить сервис, модель, лимиты, глоссарий и промпт.
5. Запустить перевод.
6. При необходимости открыть `Валидатор` и довести результат вручную.
7. Если нужен формат для публикации, использовать `EPUB -> Rulate MD`.
8. Если нужен сайт RanobeLib, использовать `RanobeLib Uploader`.

## Какие файлы появляются в проекте

Во время работы рядом с переводом могут появляться:

- `translation_map.json` — карта проекта и служебные связи между файлами;
- `queue_snapshot.db` — сохранённое состояние очереди;
- переведённые главы;
- файлы глоссария и промежуточные служебные данные.

Пользователю не нужно редактировать их вручную без необходимости.

## Важные привычки

- Всегда создавайте отдельную папку проекта под каждую книгу.
- Не смешивайте несколько книг в одной рабочей папке.
- Перед крупными изменениями в очереди делайте резервную копию очереди.
- Если перевод делается долго и нестабильно, сначала уменьшайте параллельность, а не только меняйте модель.
- Если перевод выглядит несогласованным, сначала проверьте глоссарий и промпт, потом уже повторно гоните главы.
