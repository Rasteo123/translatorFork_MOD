# GitHub Release Draft

Suggested tag: `v10.5.18`
Suggested title: `translatorFork_MOD v10.5.18`
Range: `v10.5.17..HEAD` (`2026-04-25`)

## Release Body

## Что нового

- Добавлен последовательный перевод глав с отдельным промптом `default_sequential_prompt.txt`: контекст предыдущих глав теперь можно передавать в следующую главу более управляемо.
- В `Gemini Reader` расширена устойчивость длинных TTS-задач: доработаны ротация ключей, возобновление, замена проблемных ключей и обработка разных форматов книг.
- Улучшен экспорт аудио/видео в `Gemini Reader`: нормализация объединённого аудио, более высокое качество итогового звука и предсказуемая обработка готовых сегментов.
- Обновлены доступные модели и провайдеры: добавлены актуальные модели DeepSeek, `Gemini Robotics ER 1.6 Preview`, автоопределение локальных моделей и запуск local-like моделей без обязательного API-ключа.
- В интерфейсе перевода добавлены сворачиваемые панели настроек, чтобы большие проекты и расширенные параметры занимали меньше места.

## Исправления

- `AI Consistency` стал устойчивее к плохим и подвисшим ключам: чанки повторяются после сбоя, а общий глоссарий проекта корректнее используется между переводом и проверкой консистентности.
- Исправлены зависания `Gemini Reader` и `WorkAscii` в отдельных сценариях долгих задач.
- Исправлен выбор ключа при тесте голоса в `Gemini Reader`.
- Исправлены лимиты вывода и значения по умолчанию для локальных моделей, а автоматическое обнаружение Ollama стало тише и не шумит в UI без необходимости.
- Исправлено зависание окна истории проектов.
- Сохранены ручные настройки размера задач, чтобы пользовательские override не сбрасывались при обновлении параметров.

## Технические изменения

- Добавлены регрессионные тесты для последовательного перевода, устойчивости `AI Consistency`, локальных API-моделей, DeepSeek, обработки ошибок, Reader audio/video export, key rotation/resume и форматных сценариев Reader.
- Обновлена конфигурация провайдеров и базовые настройки моделей для локальных, Gemini, DeepSeek, HuggingFace и OpenRouter обработчиков.
- Доработаны подсистемы `task_manager`, `prompt_builder`, `translation_engine` и обработка ошибок для новых sequential/consistency сценариев.
